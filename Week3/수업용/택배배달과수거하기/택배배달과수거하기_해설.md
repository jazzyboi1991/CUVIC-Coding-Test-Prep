## 1. 문제 분석

### 알고리즘 유형
- **그리디 (Greedy, 탐욕법)**
- **투 포인터 (Two Pointers) 또는 스택**

### 문제의 핵심 및 난이도
- 배달해야 할 상자와 수거해야 할 빈 상자가 있을 때, 트럭이 모든 작업을 완수하기 위한 **최소 왕복 누적 이동 거리**를 계산해야 합니다.
- **그리디 전략**: 먼 곳에 있는 집일수록 무조건 가야만 하고, 가장 멀리 있는 집에 갈 때 **가는 길에 배달할 수 있는 상자들을 최대한 끝에서부터 배달**하고, **오는 길에 수거할 수 있는 상자들을 최대한 끝에서부터 수거**해야 합니다. 즉, 언제나 **가장 먼 거리의 집부터 처리하는 것이 최적의 해**를 보장합니다.
- 먼 곳부터 처리할 때, 배달/수거 능력을 누적해 나가는 그리디 기법을 사용하면 아주 깔끔하고 빠른 계산이 가능해집니다.
- 난이도는 프로그래머스 **Level 2**로, 꼼꼼하게 끝에서부터 계산하는 예외 처리 및 시간 복잡도를 $O(N)$으로 가져가는 최적화 능력이 요구되는 까다로운 그리디 문제입니다.

## 2. 단계별 풀이 전략

이해하기 쉬운 직관적인 상태 누적(Accumulation) 그리디 기법을 적용합니다.

### [1단계] 배열 뒤에서부터 순회하기
- 가장 멀리 있는 집이 트럭이 방문해야 할 최대 거리를 결정합니다.
- 따라서 $n-1$번째 집(가장 먼 집)부터 $0$번째 집(가장 가까운 집)까지 **거꾸로** 순회합니다.

### [2단계] 배달 및 수거 누적 용량 관리
- 현재 집에 배달해야 할 양(`deliveries[i]`)과 수거해야 할 양(`pickups[i]`)을 각각의 누적 변수(`d_cap`, `p_cap`)에서 차감합니다.
- 만약 이 누적 변수들이 음수가 된다면(즉, 현재 집에 갖다 줘야 하거나 가져와야 할 박스가 쌓여 있다면), 트럭이 물류창고를 왕복해야 한다는 의미입니다.
- **왕복 연산**:
  - 부족한 용량을 메우기 위해 몇 번 왕복해야 하는지 몫 연산으로 계산합니다.
  - 왕복 횟수 `cnt`를 구하고, 누적 용량에 `cnt * cap`만큼 보충해 줍니다.
  - 전체 누적 거리(`answer`)에 **`(현재 집의 거리) * 2 * 왕복 횟수`**를 더해 줍니다. (거리 = `i + 1`)
- 이 과정을 배열의 처음까지 반복합니다.

## 3. 코드 구현 (Python, C++, Java)

### 🐍 Python 코드
사용자에게 직접 입력(cap, n, deliveries, pickups)을 받아 동작 결과를 확인해볼 수 있는 대화형 스크립트입니다.

```python
def solution(cap, n, deliveries, pickups):
    # 최소 누적 이동 거리를 저장할 변수
    answer = 0
    
    # 배달 및 수거를 위한 누적 잔여 수량 관리 변수
    # d_cap: 현재 트럭의 여유 배달 허용치 (양수면 배달 가능, 음수면 물류창고 왕복 필요)
    # p_cap: 현재 트럭의 여유 수거 허용치
    d_cap = 0
    p_cap = 0
    
    # 제일 먼 곳(뒤에서부터)부터 그리디하게 해결
    for i in range(n - 1, -1, -1):
        # 현재 집에 배달해야 할 상자와 수거할 상자 개수를 차감
        d_cap -= deliveries[i]
        p_cap -= pickups[i]
        
        # 만약 배달이나 수거 둘 중 하나라도 용량이 부족해져 음수가 되면, 왕복이 필요한 상태
        cnt = 0
        while d_cap < 0 or p_cap < 0:
            # 트럭이 한 번 더 왕복할 때마다 cap만큼의 배달/수거 능력이 생김
            d_cap += cap
            p_cap += cap
            cnt += 1  # 왕복 횟수 카운트
            
        # 왕복한 횟수만큼 거리 누적 (i + 1은 1부터 시작하는 실제 거리)
        answer += (i + 1) * 2 * cnt
        
    return answer

# --- 직접 실행하여 결과를 검증할 수 있는 Main부 ---
if __name__ == "__main__":
    print("=== [Python] 택배 배달과 수거하기 실행기 ===")
    try:
        cap = int(input("트럭의 최대 적재 용량(cap)을 입력하세요: "))
        n = int(input("집의 개수(n)를 입력하세요: "))
        
        del_input = input("각 집의 배달량을 공백 구분으로 입력하세요: ")
        deliveries = list(map(int, del_input.split()))
        
        pic_input = input("각 집의 수거량을 공백 구분으로 입력하세요: ")
        pickups = list(map(int, pic_input.split()))
        
        result = solution(cap, n, deliveries, pickups)
        print(f"👉 최소 이동 거리: {result}")
    except ValueError:
        print("올바른 형태의 정수를 입력하세요.")
```

### 🔵 C++ 코드
효율적인 $O(N)$ 반복 구조와 표준 `cin` 입력을 매끄럽게 연결한 완결형 C++ 코드입니다.

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// solution 함수: 최소 이동 거리를 계산합니다.
long long solution(int cap, int n, vector<int> deliveries, vector<int> pickups) {
    long long answer = 0; // 누적 거리가 int 범위를 초과할 수 있으므로 long long 타입 사용
    
    int d_cap = 0; // 배달 가능한 누적 용량
    int p_cap = 0; // 수거 가능한 누적 용량
    
    // 가장 먼 집(n-1)부터 거꾸로 방문하여 그리디하게 처리합니다.
    for (int i = n - 1; i >= 0; i--) {
        d_cap -= deliveries[i];
        p_cap -= pickups[i];
        
        int cnt = 0; // 현재 위치에서 왕복해야 할 횟수
        
        // 배달 또는 수거 용량이 음수라는 것은 물류창고에서 트럭이 더 와야 한다는 의미
        while (d_cap < 0 || p_cap < 0) {
            d_cap += cap;
            p_cap += cap;
            cnt++;
        }
        
        // 왕복 횟수(cnt)가 있으면, 해당 집의 거리(i + 1) * 2 * cnt 만큼 누적 거리를 합산
        if (cnt > 0) {
            answer += (long long)(i + 1) * 2 * cnt;
        }
    }
    
    return answer;
}

// --- 직접 실행하여 결과를 검증할 수 있는 Main 함수 ---
int main() {
    cout << "=== [C++] 택배 배달과 수거하기 실행기 ===" << endl;
    
    int cap, n;
    cout << "트럭 최대 용량(cap)과 집의 개수(n)를 입력하세요 (공백 구분): ";
    if (!(cin >> cap >> n)) return 0;
    
    vector<int> deliveries(n);
    cout << n << "개의 배달량을 순서대로 입력하세요: ";
    for (int i = 0; i < n; i++) {
        cin >> deliveries[i];
    }
    
    vector<int> pickups(n);
    cout << n << "개의 수거량을 순서대로 입력하세요: ";
    for (int i = 0; i < n; i++) {
        cin >> pickups[i];
    }
    
    long long result = solution(cap, n, deliveries, pickups);
    cout << "👉 최소 이동 거리: " << result << endl;
    
    return 0;
}
```

### ☕ Java 코드
Java에서 대용량 데이터를 처리할 때 입출력 오버헤드를 막는 스캐너 구조와 `long` 자료형 안정성을 높인 소스 코드입니다.

```java
// 표준 입출력을 수행하기 위한 Scanner 클래스 임포트
import java.util.Scanner;

public class Main {
    
    // solution 메소드: 최소 이동거리를 long 타입으로 정밀하게 연산합니다.
    public static long solution(int cap, int n, int[] deliveries, int[] pickups) {
        long answer = 0; // 최종 왕복 거리가 매우 클 수 있으므로 long 사용
        
        int d_cap = 0; // 누적 배달 가능 상자 수
        int p_cap = 0; // 누적 수거 가능 상자 수
        
        // 먼 집부터 거꾸로 탐색하며 배달/수거 능력을 차감 및 왕복 갱신합니다.
        for (int i = n - 1; i >= 0; i--) {
            d_cap -= deliveries[i];
            p_cap -= pickups[i];
            
            int cnt = 0; // 현재 집 방문을 위해 추가로 필요한 왕복 횟수
            
            // 배달 용량 또는 수거 용량이 부족하다면
            while (d_cap < 0 || p_cap < 0) {
                d_cap += cap; // 트럭 한 대 왕복으로 cap만큼 보충
                p_cap += cap;
                cnt++;
            }
            
            // 물류창고(0)에서 해당 집까지의 거리(i+1) 왕복 비용 누적
            if (cnt > 0) {
                answer += (long)(i + 1) * 2 * cnt;
            }
        }
        
        return answer;
    }

    // --- 직접 실행하여 결과를 검증할 수 있는 Main 메소드 ---
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.println("=== [Java] 택배 배달과 수거하기 실행기 ===");
        
        System.out.print("트럭의 최대 용량(cap)을 입력하세요: ");
        int cap = scanner.nextInt();
        
        System.out.print("집의 개수(n)를 입력하세요: ");
        int n = scanner.nextInt();
        
        int[] deliveries = new int[n];
        System.out.print(n + "개의 배달량을 순서대로 입력하세요 (공백 구분): ");
        for (int i = 0; i < n; i++) {
            deliveries[i] = scanner.nextInt();
        }
        
        int[] pickups = new int[n];
        System.out.print(n + "개의 수거량을 순서대로 입력하세요 (공백 구분): ");
        for (int i = 0; i < n; i++) {
            pickups[i] = scanner.nextInt();
        }
        
        long result = solution(cap, n, deliveries, pickups);
        System.out.println("👉 최소 이동 거리: " + result);
        
        scanner.close(); // 자원 회수
    }
}
```

## 4. 핵심 포인트 및 주의 사항

1. **오버플로우 방지 (`long long` / `long` 사용)**:
   - 집의 개수 $N$은 최대 100,000개입니다.
   - 각 집의 거리는 최대 100,000이고 배달 상자는 50개까지 쌓일 수 있어 최악의 경우 총 누적 이동 거리는 32비트 정수형(`int`) 범위인 21억을 훌쩍 넘길 수 있습니다.
   - 따라서 C++에서는 `long long`, Java에서는 `long` 형을 사용하여 거리 계산을 처리해야 오버플로우로 인한 오답을 방지할 수 있습니다.

2. **시간 복잡도**:
   - 가장 먼 집부터 순회하면서 각 집에 대해 필요한 배달 및 수거 물량을 계산하므로 루프는 $N$번 돕니다.
   - 부족한 용량을 메우기 위한 내부 `while` 루프는 상수가 크지 않고 최악의 경우에도 몇 번의 사칙연산만 수행하므로 사실상 $O(1)$에 가깝습니다.
   - 전체 시간 복잡도는 **$O(N)$**이 되어 $N = 100,000$인 복잡한 테스트 케이스도 2~5ms 내외로 극도로 빠르게 패스합니다.

3. **그리디(Greedy)의 정당성**:
   - 거리가 가장 먼 곳에 있는 집은 어떻게 돌아가더라도 무조건 트럭이 방문해야 하는 최대 거리를 정의합니다.
   - 한 번 그 거리까지 트럭이 도달했다면, 돌아오는 길에 트럭의 빈 칸을 채워 수거하고, 가는 길에 물량을 실어서 배달하는 것을 **동시에 해결**하는 것이 개별적으로 이동하는 것보다 언제나 거리를 덜 낭비하는 최적의 방법입니다.
