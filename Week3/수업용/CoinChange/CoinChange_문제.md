# LeetCode 322. Coin Change (동전 교환)

## 문제 설명

다양한 동전들의 액면가를 나타내는 정수 배열 `coins`와, 총액을 나타내는 정수 `amount`가 주어집니다.

이 총액을 만들기 위해 필요한 **가장 적은 동전의 개수**를 반환하세요. 만약 어떤 동전의 조합으로도 그 총액을 만들 수 없다면 `-1`을 반환하세요.

각 종류의 동전은 **무한히 많은 개수**를 사용할 수 있다고 가정합니다.

---

## 제한사항

- $1 \le \text{coins.length} \le 12$
- $1 \le \text{coins}[i] \le 2^{31} - 1$
- $0 \le \text{amount} \le 10^4$

---

## 입출력 예

| coins | amount | return |
| :--- | :--- | :--- |
| `[1, 2, 5]` | 11 | 3 |
| `[2]` | 3 | -1 |
| `[1]` | 0 | 0 |

---

## 입출력 예 설명

### 입출력 예 #1
- 11원을 만들기 위해 `5원 동전 2개`와 `1원 동전 1개`를 조합하면 총 3개의 동전이 필요합니다. (5 + 5 + 1 = 11) 이보다 적은 수의 동전 조합은 존재하지 않습니다.

### 입출력 예 #2
- 2원 동전만을 무한정 사용할 수 있는 상황에서는 어떤 조합으로도 홀수인 3원을 만들 수 없으므로 `-1`을 반환합니다.

### 입출력 예 #3
- 만들어야 하는 액수가 0원이면 동전이 전혀 필요하지 않으므로 `0`을 반환합니다.
