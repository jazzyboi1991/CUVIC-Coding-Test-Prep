# Summer/Winter Coding(~2018) - 예산 해설

본 문서는 **Summer/Winter Coding(~2018)**에 출제된 **예산** 문제의 알고리즘 분석, 상세 풀이 전략 및 3개 언어(Python, C++, Java)의 모범 답안과 실행 코드를 제공합니다.

---

## 1. 문제 분석

### 알고리즘 유형
- **그리디 (Greedy, 탐욕법)**
- **정렬 (Sorting)**

### 문제의 핵심 및 난이도
- 이 문제는 전체 제한된 예산 안에서 **최대한 많은 부서**를 지원하는 것이 목표입니다.
- 최대한 많은 부서를 지원하기 위해서는 **신청 금액이 적은 부서부터 우선적으로 지원**해야 합니다. 즉, "매 순간 가장 비용이 적게 드는 선택을 하는 것"이 전체 최적의 해로 이어지는 전형적인 **그리디(Greedy)** 구조를 띄고 있습니다.
- 난이도는 프로그래머스 **Level 1**로, 그리디 알고리즘의 기초 개념을 훈련하기에 매우 좋은 문제 입문용 문제입니다.

---

## 2. 단계별 풀이 전략

누구나 쉽게 이해할 수 있도록 로직의 흐름을 단계별로 파악해 보겠습니다.

### [1단계] 부서별 신청 금액 정렬하기
- 예산이 한정되어 있을 때 많은 부서를 돕는 최선의 방법은 가장 적은 돈을 요구하는 부서부터 처리하는 것입니다.
- 따라서 신청 금액이 담긴 배열 `d`를 **오름차순(작은 순서대로)**으로 정렬합니다.
- 예시: `[1, 3, 2, 5, 4]` 정렬 후 ➡️ `[1, 2, 3, 4, 5]`

### [2단계] 순차적으로 예산에서 차감하기
- 정렬된 신청 금액을 앞에서부터 하나씩 확인하며 예산(`budget`)에서 차감합니다.
- 차감하기 전에 **남은 예산이 현재 신청한 금액보다 크거나 같은지** 확인합니다.
- 만약 지원해 줄 수 있다면:
  1. 예산(`budget`)에서 해당 금액을 뺍니다.
  2. 지원한 부서의 수(`answer`)를 1만큼 증가시킵니다.
- 만약 남은 예산이 현재 부서의 신청 금액보다 적다면:
  - 현재 부서 이후의 부서들은 무조건 신청 금액이 더 크거나 같으므로(정렬되어 있기 때문), 더 이상 지원할 수 없습니다. 즉시 반복을 종료합니다.

### [3단계] 최종 결과 반환
- 예산이 다 소진되거나 모든 부서를 순회하고 난 뒤 계산된 지원 부서의 수(`answer`)를 반환합니다.

## 3. 코드 구현 (Python, C++, Java)

### 🐍 Python 코드
터미널에서 직접 부서 신청 금액 배열과 예산을 입력해 테스트할 수 있는 전체 실행 코드입니다.

```python
def solution(d, budget):
    # 지원해 준 부서의 개수를 저장할 변수
    answer = 0
    
    # 1. 신청 금액이 적은 부서부터 지원하기 위해 오름차순 정렬
    d.sort()
    
    # 2. 정렬된 신청 금액을 하나씩 확인
    for cost in d:
        # 남은 예산으로 현재 부서의 신청 금액을 지원할 수 있는지 확인
        if budget >= cost:
            budget -= cost  # 예산에서 차감
            answer += 1     # 지원한 부서 수 증가
        else:
            # 예산이 부족하면 이후 부서도 지원할 수 없으므로 루프 종료 (그리디)
            break
            
    return answer

# --- 직접 실행하여 결과를 검증할 수 있는 Main부 ---
if __name__ == "__main__":
    print("=== [Python] 예산 문제 실행기 ===")
    try:
        # 공백으로 구분하여 부서별 신청 금액 입력 받기
        user_input = input("부서별 신청 금액을 공백으로 구분하여 입력하세요 (예: 1 3 2 5 4): ")
        d = list(map(int, user_input.split()))
        
        # 총 예산 입력 받기
        budget = int(input("전체 예산을 입력하세요 (예: 9): "))
        
        # solution 함수 실행 및 결과 출력
        result = solution(d, budget)
        print(f"👉 지원할 수 있는 최대 부서 수: {result}개")
    except ValueError:
        print("올바른 형식의 숫자를 입력해주세요.")
```

### 🔵 C++ 코드
C++ 표준 입출력 스트림을 통해 사용자가 직접 대화형으로 값을 입력하여 실행해 볼 수 있는 구조입니다.

```cpp
#include <iostream>
#include <vector>
#include <algorithm> // sort 함수 사용을 위한 헤더

using namespace std;

// solution 함수: 최대 지원 가능한 부서 개수를 계산합니다.
int solution(vector<int> d, int budget) {
    int answer = 0; // 지원한 부서의 수를 기록할 변수
    
    // 1. 신청 금액을 오름차순으로 정렬합니다.
    sort(d.begin(), d.end());
    
    // 2. 신청 금액이 적은 부서부터 순차적으로 지원합니다.
    for (int i = 0; i < d.size(); i++) {
        // 남은 예산이 현재 부서의 신청 금액 이상인 경우 지원 가능
        if (budget >= d[i]) {
            budget -= d[i]; // 예산에서 신청 금액만큼 차감
            answer++;       // 지원 부서 카운트 증가
        } else {
            // 현재 신청 금액보다 예산이 적다면, 이후 부서는 정렬되어 있어 더 크므로
            // 더 이상 탐색하지 않고 반복문을 탈출합니다 (그리디).
            break;
        }
    }
    
    return answer; // 계산된 최대 부서 수 반환
}

// --- 직접 실행하여 결과를 검증할 수 있는 Main 함수 ---
int main() {
    cout << "=== [C++] 예산 문제 실행기 ===" << endl;
    
    int n;
    cout << "부서의 개수를 입력하세요: ";
    if (!(cin >> n)) return 0;
    
    vector<int> d(n);
    cout << n << "개의 부서 신청 금액을 순서대로 입력하세요 (공백 구분): ";
    for (int i = 0; i < n; i++) {
        cin >> d[i];
    }
    
    int budget;
    cout << "전체 예산을 입력하세요: ";
    cin >> budget;
    
    int result = solution(d, budget);
    cout << "👉 지원할 수 있는 최대 부서 수: " << result << "개" << endl;
    
    return 0;
}
```

### ☕ Java 코드
Java 표준 `Scanner` 객체를 활용하고 컴파일 에러를 방지하도록 와일드카드(`import java.util.*;`) 대신 상세 주석과 함께 개별 클래스를 명시적으로 임포트하여 구현했습니다.

```java
// 표준 입출력을 통해 사용자로부터 데이터를 입력받기 위한 Scanner 클래스 임포트
import java.util.Scanner;
// 동적 배열 구조를 편리하게 사용하기 위한 ArrayList 클래스 임포트
import java.util.ArrayList;
// 배열 또는 컬렉션을 편리하게 정렬하기 위한 Arrays 클래스 임포트
import java.util.Arrays;

public class Main {
    
    // solution 메소드: 최대 지원 가능한 부서 개수를 연산합니다.
    public static int solution(int[] d, int budget) {
        int answer = 0; // 지원 완료한 부서 개수를 누적할 변수
        
        // 1. 적은 금액을 신청한 부서부터 만족시키기 위해 오름차순으로 정렬합니다.
        Arrays.sort(d);
        
        // 2. 정렬된 신청 금액 배열을 순차적으로 탐색합니다.
        for (int i = 0; i < d.length; i++) {
            // 현재 부서의 요청을 지원할 수 있는 잔여 예산이 있는지 비교
            if (budget >= d[i]) {
                budget -= d[i]; // 예산에서 지원금만큼 차감
                answer++;       // 지원 부서 수 1개 추가
            } else {
                // 잔여 예산이 현재 필요금액보다 부족한 경우
                // 오름차순 정렬 상태이므로 다음 부서들도 모두 지원 불가하므로 루프를 종료합니다.
                break;
            }
        }
        
        return answer; // 최대 지원 가능 부서 수 반환
    }

    // --- 직접 실행하여 결과를 검증할 수 있는 Main 메소드 ---
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.println("=== [Java] 예산 문제 실행기 ===");
        
        System.out.print("부서의 개수를 입력하세요: ");
        int n = scanner.nextInt();
        
        int[] d = new int[n];
        System.out.print(n + "개의 부서 신청 금액을 순서대로 입력하세요 (공백 구분): ");
        for (int i = 0; i < n; i++) {
            d[i] = scanner.nextInt();
        }
        
        System.out.print("전체 예산을 입력하세요: ");
        int budget = scanner.nextInt();
        
        int result = solution(d, budget);
        System.out.println("👉 지원할 수 있는 최대 부서 수: " + result + "개");
        
        scanner.close(); // 자원 반납
    }
}
```

## 4. 핵심 포인트 및 주의 사항

1. **최적 부분 구조(Optimal Substructure) 만족**:
   - 전체 문제의 최적 해(최대 부서 지원)가 부분 문제의 최적 해(가장 비용이 적은 부서부터 선택)로 구성되므로, 그리디 알고리즘을 완벽하게 적용할 수 있습니다.
2. **제한 사항 분석**:
   - 부서의 수 `d`의 길이는 최대 100입니다.
   - 퀵 정렬(정렬 알고리즘)의 평균 시간 복잡도는 $O(N \log N)$ 이며, $N = 100$ 일 때 약 700번의 연산만으로 완료됩니다.
   - 이후 한 번의 선형 탐색($O(N)$)으로 답을 도출하므로, 최악의 시나리오에서도 극도로 빠른 속도(1ms 미만)로 통과할 수 있습니다.
3. **자료형의 안정성**:
   - 신청 금액은 최대 100,000 이하이며, `budget`은 최대 10,000,000 이하이므로, 표준 32비트 정수형(`int`) 범위 내에서 안전하게 오버플로우 걱정 없이 연산이 가능합니다.
