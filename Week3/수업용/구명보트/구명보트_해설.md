# 탐욕법(Greedy) - 구명보트 해설

본 문서는 **탐욕법(Greedy)** 유형의 대표적인 문제인 **구명보트**의 상세한 알고리즘 분석, 풀이 전략 및 3개 언어(Python, C++, Java)의 모범 답안과 실행 코드를 제공합니다.

---

## 1. 문제 분석

### 알고리즘 유형
- **그리디 (Greedy, 탐욕법)**
- **정렬 (Sorting)**
- **투 포인터 (Two Pointers)**

### 문제의 핵심 및 난이도
- 이 문제는 무인도에 갇힌 사람들을 구명보트를 활용해 탈출시킬 때, **필요한 구명보트 개수의 최솟값**을 구하는 문제입니다.
- 구명보트의 가장 핵심적인 제약조건은 **"한 번에 최대 2명씩 밖에 탈 수 없다"**는 점입니다. 이 제약 조건이 문제를 단순화시켜 주는 핵심 열쇠입니다.
- 보트를 가장 적게 쓰려면 가능한 2명씩 짝을 지어 태워 보내야 합니다.
- **그리디 전략**: 가장 무거운 사람을 태울 때, 이 사람과 **함께 탈 수 있는 가장 가벼운 사람**을 찾아서 짝을 짓는 것이 항상 최선의 선택이 됩니다.
- 난이도는 프로그래머스 **Level 2**로, 정렬 후 양방향에서 좁혀 들어오는 **투 포인터(Two Pointers)** 기법을 결합하여 효율적으로 풀어야 효율성 테스트를 통과할 수 있습니다.

---

## 2. 단계별 풀이 전략

누구나 논리 흐름을 따라갈 수 있도록 단계적으로 분석합니다.

### [1단계] 몸무게 정렬하기
- 사람들의 몸무게를 담은 배열 `people`을 **오름차순(가벼운 사람 ➡️ 무거운 사람)**으로 정렬합니다.
- 예시: `[70, 50, 80, 50]` 정렬 후 ➡️ `[50, 50, 70, 80]`

### [2단계] 양방향 포인터 설정하기
- 두 명씩 짝을 짓기 위해 두 개의 포인터를 설정합니다.
  - `light`: 가장 가벼운 사람을 가리키는 포인터 (배열의 시작점인 `0`)
  - `heavy`: 가장 무거운 사람을 가리키는 포인터 (배열의 끝점인 `len(people) - 1`)

### [3단계] 짝 지어 보트 태우기 (그리디 + 투 포인터)
- `light <= heavy`인 동안 반복하여 사람들을 태웁니다.
- **체크포인트**: 가장 가벼운 사람(`people[light]`)과 가장 무거운 사람(`people[heavy]`)의 몸무게 합을 제한 무게(`limit`)와 비교합니다.
  - **합이 `limit` 이하인 경우 (두 명 탑승 가능)**:
    - 가장 가벼운 사람도 보트에 함께 탈 수 있으므로 다음 가벼운 사람을 가리키도록 `light` 포인터를 오른쪽으로 이동시킵니다 (`light += 1`).
    - 무거운 사람도 탔으므로 `heavy` 포인터를 왼쪽으로 이동시킵니다 (`heavy -= 1`).
  - **합이 `limit`를 초과하는 경우 (무거운 사람만 탑승 가능)**:
    - 가장 가벼운 사람과 조합해도 보트에 같이 탈 수 없을 만큼 무거운 사람이므로, 이 무거운 사람은 무조건 보트에 혼자 타야 합니다.
    - 무거운 사람만 탑승 처리하므로 `heavy` 포인터만 왼쪽으로 이동시킵니다 (`heavy -= 1`).
- 매 반복마다 보트가 1대씩 소모되므로 보트 개수(`answer`)를 1씩 증가시킵니다.

## 3. 코드 구현 (Python, C++, Java)

### 🐍 Python 코드
사용자로부터 사람들의 몸무게 배열과 보트 무게 제한을 입력받아 실행 결과를 곧바로 검증할 수 있는 전체 실행 코드입니다.

```python
def solution(people, limit):
    # 사용된 구명보트 개수
    answer = 0
    
    # 1. 효율적인 짝 짓기를 위해 몸무게 오름차순 정렬
    people.sort()
    
    # 2. 투 포인터 설정
    light = 0                  # 가장 가벼운 사람의 인덱스
    heavy = len(people) - 1    # 가장 무거운 사람의 인덱스
    
    # 3. 모든 사람이 탈출할 때까지 반복
    while light <= heavy:
        # 가장 가벼운 사람과 가장 무거운 사람의 합이 limit 이하인지 비교
        if people[light] + people[heavy] <= limit:
            # 두 명이 같이 탈 수 있는 경우
            light += 1  # 가벼운 사람 탑승 처리 (다음 가벼운 사람으로 이동)
            
        # 가장 무거운 사람은 무조건 보트에 탑승 처리 (혼자 타든, 가벼운 사람과 타든)
        heavy -= 1      # 무거운 사람 탑승 처리 (다음 무거운 사람으로 이동)
        answer += 1     # 구명보트 1대 증가
        
    return answer

# --- 직접 실행하여 결과를 검증할 수 있는 Main부 ---
if __name__ == "__main__":
    print("=== [Python] 구명보트 문제 실행기 ===")
    try:
        user_input = input("사람들의 몸무게를 공백으로 구분하여 입력하세요 (예: 70 50 80 50): ")
        people = list(map(int, user_input.split()))
        
        limit = int(input("구명보트의 무게 제한을 입력하세요 (예: 100): "))
        
        result = solution(people, limit)
        print(f"👉 필요한 최소 구명보트 개수: {result}대")
    except ValueError:
        print("올바른 정수들을 입력해주세요.")
```

### 🔵 C++ 코드
C++ 표준 STL의 `sort`와 `vector`를 활용하고, 대화형 콘솔 입력 기능을 제공하는 전체 C++ 소스 코드입니다.

```cpp
#include <iostream>
#include <vector>
#include <algorithm> // sort 함수 사용을 위한 헤더

using namespace std;

// solution 함수: 구명보트 개수의 최솟값을 연산합니다.
int solution(vector<int> people, int limit) {
    int answer = 0; // 필요한 보트 수
    
    // 1. 몸무게 오름차순 정렬
    sort(people.begin(), people.end());
    
    // 2. 투 포인터 변수 정의
    int light = 0;                  // 가장 가벼운 사람을 가리키는 포인터
    int heavy = people.size() - 1;  // 가장 무거운 사람을 가리키는 포인터
    
    // 3. 모든 인원이 탈 때까지 양 끝에서 포인터를 좁혀옵니다.
    while (light <= heavy) {
        // 가장 가벼운 사람과 무거운 사람의 무게 합이 제한 이하인지 검사
        if (people[light] + people[heavy] <= limit) {
            light++; // 같이 탈 수 있으므로 가벼운 사람 포인터 증가
        }
        // 무거운 사람은 무조건 이번 보트에 탑승하게 되므로 무거운 사람 포인터 감소
        heavy--;
        answer++; // 보트 사용 횟수 카운트
    }
    
    return answer;
}

// --- 직접 실행하여 결과를 검증할 수 있는 Main 함수 ---
int main() {
    cout << "=== [C++] 구명보트 문제 실행기 ===" << endl;
    
    int n;
    cout << "대피할 사람들의 수를 입력하세요: ";
    if (!(cin >> n)) return 0;
    
    vector<int> people(n);
    cout << n << "명의 몸무게를 공백으로 구분하여 입력하세요: ";
    for (int i = 0; i < n; i++) {
        cin >> people[i];
    }
    
    int limit;
    cout << "구명보트 무게 제한을 입력하세요: ";
    cin >> limit;
    
    int result = solution(people, limit);
    cout << "👉 필요한 최소 구명보트 개수: " << result << "대" << endl;
    
    return 0;
}
```

### ☕ Java 코드
성능 극대화를 위해 `Arrays.sort`를 사용하고 표준 임포트와 대화형 입력을 명확히 작성한 완결된 Java 소스 코드입니다.

```java
// 콘솔 입력을 안전하게 구현하기 위해 Scanner 클래스를 수동 임포트합니다.
import java.util.Scanner;
// 배열 정렬 라이브러리인 Arrays 클래스를 명시적으로 임포트합니다.
import java.util.Arrays;

public class Main {
    
    // solution 메소드: 구명보트의 최솟값을 연산합니다.
    public static int solution(int[] people, int limit) {
        int answer = 0; // 구명보트 누적 개수
        
        // 1. 오름차순 정렬하여 가장 가벼운 사람과 무거운 사람을 쉽게 매칭할 수 있도록 합니다.
        Arrays.sort(people);
        
        // 2. 투 포인터(인덱스 변수)를 선언합니다.
        int light = 0;                  // 첫 번째 원소 (가장 가벼운 사람)
        int heavy = people.length - 1;  // 마지막 원소 (가장 무거운 사람)
        
        // 3. 포인터가 서로 엇갈리기 전까지 반복 탐색을 수행합니다.
        while (light <= heavy) {
            // 가장 가벼운 사람과 무거운 사람의 무게 합이 제한을 초과하지 않는 경우
            if (people[light] + people[heavy] <= limit) {
                light++; // 가벼운 사람도 이번 보트에 함께 탑승 처리
            }
            
            // 무거운 사람은 합이 넘든 안 넘든 이번 보트에 타고 나가야 하므로 탑승 처리
            heavy--;
            answer++; // 보트 개수 1대 추가
        }
        
        return answer; // 최종 필요한 보트 수 반환
    }

    // --- 직접 실행하여 결과를 검증할 수 있는 Main 메소드 ---
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.println("=== [Java] 구명보트 문제 실행기 ===");
        
        System.out.print("대피할 사람들의 수를 입력하세요: ");
        int n = scanner.nextInt();
        
        int[] people = new int[n];
        System.out.print(n + "명의 몸무게를 공백으로 구분하여 입력하세요: ");
        for (int i = 0; i < n; i++) {
            people[i] = scanner.nextInt();
        }
        
        System.out.print("구명보트 무게 제한을 입력하세요: ");
        int limit = scanner.nextInt();
        
        int result = solution(people, limit);
        System.out.println("👉 필요한 최소 구명보트 개수: " + result + "대");
        
        scanner.close(); // 자원 해제
    }
}
```

## 4. 핵심 포인트 및 주의 사항

1. **시간 복잡도 최적화**:
   - 무인도에 갇힌 사람의 수 $N$은 최대 50,000명입니다.
   - 만약 모든 쌍을 비교하는 완전 탐색($O(N^2)$) 기법을 사용하면 최대 $2,500,000,000$번의 연산이 일어나 시간 초과를 받게 됩니다.
   - 몸무게 정렬에 $O(N \log N)$이 소요되고, 이후 양쪽 끝에서 좁혀 들어오는 투 포인터 선형 탐색이 $O(N)$만큼 소요됩니다.
   - 전체 시간 복잡도는 **$O(N \log N)$**이 되어 50,000개 원소를 처리하는 데 약 10ms 이하의 쾌적한 속도로 효율성 테스트를 완벽히 통과할 수 있습니다.

2. **최적화 증명 (왜 가장 가벼운 사람과 무거운 사람을 짝짓는가?)**:
   - 임의의 무거운 사람 $H$가 있을 때, 보트에 같이 태울 수 있는 가장 최선의 짝은 함께 타도 `limit`를 넘지 않는 이들 중 **가장 가벼운 사람 $L$**입니다.
   - 만약 $H$가 가장 가벼운 $L$과도 보트를 함께 타지 못한다면, 그 어떤 다른 누구와도 탈 수 없으므로 반드시 **혼자** 타야 합니다.
   - 이러한 논리적 일관성을 보장하기 때문에 그리디 접근법이 정당함을 알 수 있습니다.
