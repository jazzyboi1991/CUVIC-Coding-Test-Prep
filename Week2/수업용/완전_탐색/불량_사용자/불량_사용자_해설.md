# [해설] 불량 사용자

## 문제 분석
이 문제는 주어진 사용자 아이디 목록(`user_id`)에서 불량 사용자 패턴 목록(`banned_id`)에 매칭될 수 있는 모든 사용자 조합의 개수를 찾는 문제입니다. 

**핵심 개념**:
- **패턴 매칭**: `*` 문자를 와일드카드로 사용하여 아이디가 패턴과 일치하는지 확인해야 합니다.
- **조합(Set)**: 제재 아이디 목록의 순서가 다르더라도 구성 요소가 같으면 동일한 경우로 취급합니다. 즉, 단순한 순열이 아닌 **집합**의 개수를 세어야 합니다.

**핵심 난이도**:
- `banned_id`가 `["fro**", "f****"]`와 같이 주어졌을 때, 하나의 `user_id`가 여러 `banned_id` 패턴에 동시에 매칭될 수 있습니다.
- 중복된 조합을 피하기 위해 선택된 사용자들의 인덱스를 정렬하거나 집합 자료형을 사용하여 중복 여부를 체크해야 합니다.

---

## 단계별 풀이 전략 (상세)

### 1. 아이디 매칭 함수 작성
- 두 문자열의 길이를 먼저 비교합니다.
- 길이가 같다면 한 글자씩 비교하며, `*`가 아니면서 서로 다른 문자가 나오면 불일치로 판정합니다.

### 2. DFS (백트래킹)를 이용한 탐색
- `banned_id` 배열의 첫 번째 패턴부터 마지막 패턴까지 하나씩 매칭되는 사용자를 선택해 나갑니다.
- **방문 처리 (`visited`)**: 이미 선택된 사용자는 다른 패턴에서 다시 선택할 수 없으므로 `visited` 배열을 통해 중복 선택을 방지합니다.
- **재귀 호출**: 현재 패턴에 매칭되는 사용자를 하나 골랐다면, 다음 패턴으로 넘어가 탐색을 이어갑니다.

### 3. 중복 조합 처리
- 모든 패턴에 대해 매칭되는 사용자를 골랐다면(기저 사례 도달), 선택된 사용자들의 인덱스 목록을 만듭니다.
- 이 목록을 **정렬**한 뒤 문자열이나 튜플 형태로 변환하여 전역 `set` 자료형에 추가합니다.
- 이렇게 하면 `{"frodo", "front"}`와 `{"front", "frodo"}`를 같은 경우로 취급할 수 있습니다.

### 4. 최종 결과 반환
- 탐색이 모두 끝난 후 `set`에 담긴 고유한 조합의 개수를 반환합니다.

## 코드 구현

### 1. Python

```python
def is_match(user, banned):
    if len(user) != len(banned):
        return False
    for i in range(len(user)):
        if banned[i] == '*':
            continue
        if banned[i] != user[i]:
            return False
    return True

def solution(user_id, banned_id):
    n = len(user_id)
    m = len(banned_id)
    visited = [False] * n
    results = set()
    
    def dfs(idx, current_set):
        if idx == m:
            # 선택된 인덱스들을 정렬하여 튜플로 만들어 세트에 추가 (중복 제거)
            results.add(tuple(sorted(current_set)))
            return
        
        for i in range(n):
            if not visited[i] and is_match(user_id[i], banned_id[idx]):
                visited[i] = True
                dfs(idx + 1, current_set + [i])
                visited[i] = False # 백트래킹
                
    dfs(0, [])
    return len(results)

# 터미널 입력 예시
if __name__ == "__main__":
    user_id = input("user_id를 입력하세요 (공백으로 구분): ").split()
    banned_id = input("banned_id를 입력하세요 (공백으로 구분): ").split()
    print(f"결과: {solution(user_id, banned_id)}")
```

### 2. C++

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <set>
#include <algorithm>

using namespace std;

bool is_match(string user, string banned) {
    if (user.length() != banned.length()) return false;
    for (int i = 0; i < user.length(); i++) {
        if (banned[i] == '*') continue;
        if (user[i] != banned[i]) return false;
    }
    return true;
}

set<set<int>> total_sets;
bool visited[8];

void dfs(int idx, set<int> current_set, vector<string>& user_id, vector<string>& banned_id) {
    if (idx == banned_id.size()) {
        total_sets.insert(current_set);
        return;
    }

    for (int i = 0; i < user_id.size(); i++) {
        if (!visited[i] && is_match(user_id[i], banned_id[idx])) {
            visited[i] = true;
            current_set.insert(i);
            dfs(idx + 1, current_set, user_id, banned_id);
            current_set.erase(i);
            visited[i] = false;
        }
    }
}

int solution(vector<string> user_id, vector<string> banned_id) {
    total_sets.clear();
    for (int i = 0; i < 8; i++) visited[i] = false;
    
    dfs(0, {}, user_id, banned_id);
    return total_sets.size();
}

int main() {
    int n, m;
    cout << "user_id의 개수를 입력하세요: ";
    cin >> n;
    vector<string> user_id(n);
    cout << "user_id들을 입력하세요 (공백으로 구분): ";
    for(int i=0; i<n; i++) cin >> user_id[i];

    cout << "banned_id의 개수를 입력하세요: ";
    cin >> m;
    vector<string> banned_id(m);
    cout << "banned_id들을 입력하세요 (공백으로 구분): ";
    for(int i=0; i<m; i++) cin >> banned_id[i];

    cout << "결과: " << solution(user_id, banned_id) << endl;
    return 0;
}
```

### 3. Java

```java
import java.util.HashSet; // 중복을 허용하지 않는 데이터 집합을 관리하기 위한 클래스입니다.
import java.util.Scanner; // 사용자로부터 입력을 받기 위한 도구입니다.

public class Solution {
    // 최종적인 불량 사용자 조합(집합)들을 저장할 공간입니다.
    // HashSet 안에 또 다른 HashSet을 넣어 중복된 조합을 자동으로 걸러냅니다.
    private HashSet<HashSet<Integer>> results = new HashSet<>();
    private boolean[] visited; // 사용자가 현재 선택되었는지 확인하는 배열입니다.

    // 사용자의 아이디가 불량 사용자 패턴(*)에 맞는지 확인하는 함수입니다.
    public boolean isMatch(String user, String banned) {
        if (user.length() != banned.length()) return false; // 길이가 다르면 매칭될 수 없습니다.
        for (int i = 0; i < user.length(); i++) {
            if (banned.charAt(i) == '*') continue; // '*'은 어떤 문자든 통과합니다.
            if (user.charAt(i) != banned.charAt(i)) return false; // 다른 문자가 있으면 실패입니다.
        }
        return true;
    }

    // 모든 패턴에 대해 가능한 사용자 조합을 찾는 깊이 우선 탐색(DFS) 함수입니다.
    public void dfs(int idx, HashSet<Integer> currentSet, String[] user_id, String[] banned_id) {
        // 기저 사례: 모든 불량 사용자 패턴에 대해 매칭되는 사용자를 다 찾았을 때
        if (idx == banned_id.length) {
            results.add(new HashSet<>(currentSet)); // 현재까지의 조합을 결과 세트에 추가합니다.
            return;
        }

        for (int i = 0; i < user_id.length; i++) {
            // 아직 선택되지 않았고, 현재 패턴과 일치하는 사용자라면
            if (!visited[i] && isMatch(user_id[i], banned_id[idx])) {
                visited[i] = true; // 선택 표시
                currentSet.add(i); // 현재 조합에 추가
                dfs(idx + 1, currentSet, user_id, banned_id); // 다음 패턴으로 넘어감
                currentSet.remove(i); // 돌아오면서 조합에서 제거 (백트래킹)
                visited[i] = false; // 선택 표시 해제
            }
        }
    }

    public int solution(String[] user_id, String[] banned_id) {
        results.clear(); // 결과 초기화
        visited = new boolean[user_id.length]; // 방문 배열 초기화
        dfs(0, new HashSet<>(), user_id, banned_id); // 탐색 시작
        return results.size(); // 발견된 고유한 조합의 개수를 반환합니다.
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in); // 입력 도구 생성
        Solution sol = new Solution();

        System.out.println("user_id를 입력하세요 (공백으로 구분):");
        String[] user_id = sc.nextLine().split("\\s+");

        System.out.println("banned_id를 입력하세요 (공백으로 구분):");
        String[] banned_id = sc.nextLine().split("\\s+");

        System.out.println("결과: " + sol.solution(user_id, banned_id));
    }
}
```

## 핵심 포인트
- **인덱스 활용**: 아이디 문자열 자체를 비교하는 것보다 인덱스(0, 1, 2...)를 사용하여 집합을 구성하는 것이 훨씬 빠르고 효율적입니다.
- **백트래킹**: 한 패턴에 대한 사용자를 고르고 다음 단계로 넘어갔다가 돌아올 때, `visited` 상태를 다시 되돌려주는 과정이 필수입니다.
- **중복 제거 전략**: `set<set<int>>` 구조를 사용하면 원소의 순서에 관계없이 고유한 조합만을 남길 수 있습니다.
