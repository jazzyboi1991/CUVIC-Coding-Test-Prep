# [2020 KAKAO BLIND RECRUITMENT] 블록 이동하기

## 문제 설명
로봇개발자 **"무지"**는 한 달 앞으로 다가온 "카카오배 로봇경진대회"에 출전하려는 로봇을 준비하고 있습니다. 준비 중인 로봇은 **`2 x 1`** 크기의 로봇으로 "무지"는 **"0"**과 **"1"**로 이루어진 **`N x N`** 크기의 지도에서 로봇을 움직여 **`(N, N)`** 위치까지 이동시키려 합니다. 로봇이 이동하는 지도는 가장 왼쪽 상단 좌표가 **`(1, 1)`**이며 로봇의 처음 위치는 **`(1, 1)`**과 **`(1, 2)`** 두 칸을 차지하고 있고 가로 방향으로 놓여있습니다.

로봇은 벽이 있는 칸 또는 지도 밖으로는 이동할 수 없습니다. 로봇은 한 번에 한 칸씩 이동할 수 있으며, 로봇이 차지하고 있는 두 칸 중 어느 한 칸이라도 **`(N, N)`** 위치에 도착하면 됩니다.

![블록이동-1](https://grepp-programmers.s3.amazonaws.com/files/production/33f5c19ba6/052d3514-5fca-4b85-82aa-0f9eaefae0a3.jpg)

### 로봇의 이동 및 회전
- 로봇은 상, 하, 좌, 우로 한 칸씩 이동할 수 있습니다.
- 로봇은 90도 회전할 수 있습니다. 단, 로봇이 차지하고 있는 두 칸 중 어느 한 칸을 축으로 삼아 회전해야 하며, 회전하는 과정에서 벽을 만나지 않아야 합니다.
- 회전 방향은 시계 방향과 반시계 방향 모두 가능합니다.
- 아래 그림은 로봇이 가로로 놓여있는 상태에서 (1, 2)를 축으로 시계 방향으로 회전하는 과정을 보여줍니다.

![블록이동-2](https://grepp-programmers.s3.amazonaws.com/files/production/edfcdf57d3/f87055df-91e5-4f47-b99a-400c54bfdf3a.jpg)

로봇이 **`(N, N)`** 위치까지 이동하는 데 걸리는 최소 시간을 return 하도록 solution 함수를 완성해 주세요.

### 제한사항
- `board`의 한 변의 길이는 5 이상 100 이하입니다.
- `board`의 원소는 0(빈 칸) 또는 1(벽)입니다.
- 로봇이 처음에 놓여 있는 칸 (1, 1), (1, 2)는 항상 0입니다.
- 로봇이 목표 지점까지 도달할 수 없는 경우는 주어지지 않습니다.

---

### [입출력 예]

**입력**
```text
5
0 0 0 1 1
0 0 0 1 0
0 1 0 1 1
1 1 0 0 1
0 0 0 0 0
```

**출력**
```text
7
```

### 입출력 예 설명
문제의 예시와 같습니다. 로봇이 이동한 경로는 생략하며, 최단 시간인 7초를 반환합니다.
