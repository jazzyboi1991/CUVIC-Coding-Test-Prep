# [LeetCode] 124. Binary Tree Maximum Path Sum

## 1. 문제 설명

이진 트리에서의 **경로(Path)**는 서열 내의 각 인접한 노드 쌍이 서로 연결된 간선을 가진 노드들의 시퀀스를 의미합니다. 노드는 시퀀스에서 **최대 한 번**만 나타날 수 있습니다. 경로는 반드시 루트(root)를 통과할 필요는 없습니다.

**경로 합(Path Sum)**은 경로에 포함된 노드 값들의 총합입니다.

이진 트리의 `root`가 주어졌을 때, 비어 있지 않은 모든 경로 중 **최대 경로 합**을 반환하세요.

## 2. 제한사항

- 트리의 노드 개수는 $1$ 이상 $30,000$ 이하입니다.
- 각 노드의 값은 $-1,000$ 이상 $1,000$ 이하입니다.

##### **[입출력 예]**

**예제 1**

![Example 1 Image](https://assets.leetcode.com/uploads/2020/10/13/exx1.jpg)

- **입력**
  - 트리 노드(nodes): `1 2 3`
- **출력**: `6`
- **설명**: 최적의 경로는 `2 -> 1 -> 3`이며, 경로 합은 $2 + 1 + 3 = 6$입니다.

**예제 2**

![Example 2 Image](https://assets.leetcode.com/uploads/2020/10/13/exx2.jpg)

- **입력**
  - 트리 노드(nodes): `-10 9 20 null null 15 7`
- **출력**: `42`
- **설명**: 최적의 경로는 `15 -> 20 -> 7`이며, 경로 합은 $15 + 20 + 7 = 42$입니다.

## 4. 입출력 예 설명

- 경로의 정의상 한 노드에서 시작해 부모로 올라갔다가 다른 자식으로 내려가는 형태가 가능합니다.
- 하지만 한 노드를 두 번 방문할 수는 없으므로, 어떤 노드를 기준으로 왼쪽 자식과 오른쪽 자식 방향으로 모두 뻗어나가는 경로는 그 노드의 부모 쪽으로는 더 이상 연장될 수 없습니다.
- 각 노드 값이 음수일 수도 있으므로, 합을 더했을 때 오히려 작아진다면 해당 경로를 포함하지 않는(0으로 처리하는) 선택도 고려해야 합니다.
